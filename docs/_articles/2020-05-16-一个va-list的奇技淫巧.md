### 需求

有时候需要使用vsscanf接收void**的指针列表，但是vsscanf接收的是va_list。

### 方法

抓耳挠腮一晚上，终于解决：
[code] 
    void vssf_pnt(char* str, char* format, ...)
    {
        va_list args;
        va_start(args, format);
    	  va_list ap = va_arg(args, va_list);
        vsscanf(str, format, ap);
        va_end(args);
    }  
    
    
    int main(void){
    void** pntList = NULL; 
    	int a,b;
    	char str1[10],str2[10];
    	float k;
    
    
    	pntList = malloc(3 * sizeof(void*));
    	pntList[0] = (void*)(&a);
    	pntList[1] = (void*)(&b);
    	
    	vssf_pnt("115200,8","%d,%d",pntList);
    	printf("a is %d, b is %d\r\n", a, b );
    	
    	pntList[0] = (void*)(&a);
    	pntList[1] = (void*)(str1);
    	pntList[2] = (void*)(&b);
    	vssf_pnt("32, apple,56","%d,%[^,],%d",pntList);
    	printf("~~~~~~\r\n%s\r\n",str1);
    	printf("a is %d, b is %d\r\n", a, b );
    	
    	free(pntList);
    }
    
[/code]

### 总结

va_list太难用了。这里用到va_arg的一个类型转换的方法，来解决问题。  
目前没看到别人用过，也许是太危险。  
在keil mdk没问题，用visual studio编译没问题。用gcc编译有问题。clang编译可通过，但结果不正确。原因是底层对于va_list的定义不同。
